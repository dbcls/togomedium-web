# AGENTS.md

## このファイルの役割

- この `AGENTS.md` には、モノリポ全体で共通するルールを書きます。
- `@packages/*/AGENTS.md` には、各プロジェクト固有の文脈や注意点を書きます。
- 作業対象のディレクトリに対応する `AGENTS.md` がある場合は、このファイルとあわせて必ず確認してください。
- ルールが競合する場合は、より作業対象に近い階層の `AGENTS.md` を優先してください。

## ドキュメント運用方針

- `AGENTS.md` には、長く使う運用ルール、判断基準、繰り返し参照する前提を書いてください。
- 一時的なタスクメモ、作業ログ、短期間で陳腐化する議論は `AGENTS.md` ではなく別ドキュメントに分けてください。
- 詳細な手順書や分量の多い補足は `docs/` などに分離し、`AGENTS.md` から参照してください。
- プロジェクト内ドキュメントを参照する場合は、Markdown リンクではなく `docs/...` のようなパス表記を使用してください。
- 各ディレクトリの `AGENTS.md` は、その配下で作業する時に迷いやすい内容を優先して簡潔に保ってください。
- 過去のやりとりを残す場合は、結論や判断理由を要約して記載し、会話ログそのものの転載は避けてください。

## プロジェクト概要

- このリポジトリは DBCLS が管理する培地データベース TogoMedium に関連する Webフロントエンド及び周辺コードを管理するモノリポです。
- 主に TypeScript、React、Vite、Vitest、Storybook、pnpm を用いて開発しています。

### Stanzaについて

- 本プロジェクトでは Stanza (TogoStanza) というWebComponentの仕組みを利用しています
  - Stanzaについては今後追記していきますが「Stanza」という仕組みがあるということをまずは覚えておいてください

## プロジェクト構成

- `@packages/api`
  - API の型・スキーマ定義を扱うパッケージです。
- `@packages/api-doc`
  - TypeSpec と Vite を使った API ドキュメント関連の旧パッケージです。現在は `@packages/api` 開発時の参照用として残しており、今後廃止予定です。
- `@packages/core`
  - 複数パッケージで共有する基礎ロジックの置き場です。
- `@packages/stanza`
  - TogoStanza ベースの WebComponent 群を扱うパッケージです。
- `@packages/storybook`
  - Storybook 実行環境と補助コードを扱うパッケージです。
- `@packages/web`
  - Vite ベースの React フロントエンドです。

## 作業開始時の手順

- まず、作業対象のディレクトリと関係するパッケージを特定してください。
- このルート `AGENTS.md` を確認したうえで、対象ディレクトリ配下に `AGENTS.md` がある場合は必ず続けて確認してください。
- `docs/agents/task-guides` や各 `AGENTS.md` で案内されている作業別ガイドが今回の作業に関係する場合は、その指示も先に確認してください。
- 実行環境で `node -v` を確認し、Node v24 系統で動いていることを確認してください。
- ローカル環境で `.nvm` が使える場合は、`source ~/.nvm/nvm.sh && nvm use >/dev/null` を使ってからコマンドを実行してください。
- クラウド環境などで `.nvm` が使えない場合でも、Node のメジャーバージョンが v24 系統であることを確認してください。
- 対象がコード変更かドキュメント変更かを先に判断し、必要な確認コマンドの範囲を決めてください。
- 作業に入る前に、不明点、影響範囲の大きい変更、依存追加の必要性が見えている場合は、先にユーザーへ確認してください。

## 開発環境

- パッケージ管理には `pnpm` を使用しています。
- npm script を実行する場合も、`npm` や `yarn` ではなく `pnpm` から実行してください。
- 本プロジェクトは Node v24 系統での実行を前提としています。作業前に `node -v` を実行し、Node のメジャーバージョンを確認してください。
- ローカル環境で `.nvm` が利用できる場合は、`source ~/.nvm/nvm.sh && nvm use >/dev/null` を実行してから各種コマンドを実行してください。
- ローカル環境で `nvm use` が利用できず、かつ Node のバージョンが v24 系統ではない場合は、作業を進めずにユーザーへ確認してください。
- クラウド環境など `.nvm` を利用しない環境でも、Node のメジャーバージョンが v24 系統であることを前提に作業してください。

### パッケージ管理について

- ユーザーから明示的な依頼がない限り、AI が npm モジュールをインストールまたはアンインストールしてはいけません。
- 新しいモジュールの追加が必要だと判断した場合は、作業を進める前にユーザーへ提案し、了承を得てください。

## コミニュケーション

- 不明な箇所・判断が必要な箇所があったら作業を止めてユーザーに確認してください
- ユーザーが JavaScript の概念について質問している場合でも、回答内でコード例を示す際は TypeScript を優先してください。

## コーディング指針

- 読みやすさとメンテナンスしやすさを重視してください。
- ひとつの関数やモジュールに責務を詰め込みすぎず、役割ごとに分割してください。
- できるだけ pure function を優先し、副作用を持つ処理は境界を意識して分離してください。
- pure function にできる処理はテストしやすい形で実装し、必要に応じてテストも追加してください。
- 詳細なコード規約や個別の例外は `docs/development/coding-guidelines.md` を参照してください。

## 作業完了時の手順

- 変更内容がコード変更か、ドキュメントのみの変更かを確認してください。
- 必要に応じて、実施した変更内容と確認結果をユーザーに共有してください。

### 動作確認と検証

- コードを変更した作業の完了後には、原則として format / lint / test を実行してください
  - pnpm run fmt
  - pnpm run lint
  - pnpm run test
- ドキュメントのみの変更では、上記の実行は不要です
- ただし、ユーザーが明示的に希望した場合は実行してください

## 作業別ガイド

- `docs/agents/task-guides` フォルダ以下に、作業別の詳細な指示をまとめています。
- 該当する作業別ガイドがある場合は、作業前に必ず確認してください。
- 各パッケージの `AGENTS.md` から個別の作業別ガイドを案内している場合もあるため、対象ディレクトリ側の指示も確認してください。
